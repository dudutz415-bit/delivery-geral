Guia Firebase

6 passos para ativar o backend do Delivery Geral

1
Criar projeto no Firebase
Conta Google + novo projeto

📋 O que voce precisa

  • Uma conta Google (Gmail)
  • 5 minutos de tempo

🚀 Passo a passo

  1. Acesse console.firebase.google.com
  2. Faca login com sua conta Google
  3. Clique em "Criar projeto"
  4. Digite o nome: delivery-geral
  5. Desative Google Analytics (ou deixe ativado)
  6. Clique em "Criar projeto" e aguarde
✅ Pronto! Voce ja tem o projeto criado. Agora vamos registrar o app.
2
Registrar app Web e copiar configuracao
O codigo que conecta seu app ao Firebase

🚀 Passo a passo

  1. No painel do Firebase, clique no icone "</>" (Adicionar app da Web)
  2. De um apelido: Delivery Geral Web
  3. Clique em "Registrar app"
  4. Voce vera um codigo assim:
const firebaseConfig = { apiKey: "AIzaSyB...", authDomain: "delivery-geral.firebaseapp.com", projectId: "delivery-geral", storageBucket: "delivery-geral.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abcdef123" };
  1. Copie esse codigo inteiro
  2. Abra o arquivo firebase-config.js no seu computador
  3. Cole substituindo o placeholder que esta la
  4. Faca o mesmo nos arquivos entregador.html, lojista.html e admin.html
⚠️ Importante: A mesma configuracao deve estar em TODOS os arquivos. Se mudar um, mude todos!
3
Ativar Authentication e Firestore
Login de usuarios e banco de dados

🔐 Ativar Authentication (login)

  1. No menu lateral do Firebase, clique em "Authentication"
  2. Clique em "Comecar"
  3. Selecione "Email/Senha" na lista de provedores
  4. Ative o primeiro toggle (Email/Senha)
  5. Clique em "Salvar"

💾 Ativar Firestore Database

  1. No menu lateral, clique em "Firestore Database"
  2. Clique em "Criar banco de dados"
  3. Escolha "Iniciar no modo de teste"
  4. Selecione a regiao southamerica-east1 (Sao Paulo) ou a mais proxima
  5. Clique em "Ativar"
💡 Dica: O "modo de teste" permite leitura/escrita livre por 30 dias. Depois voce publica as regras de seguranca do arquivo firestore.rules.
4
Configurar Notificacoes Push (FCM)
Avisos no celular para cliente e entregador

🚀 Passo a passo

  1. No Firebase Console, clique na engrenagem (Configuracoes do projeto)
  2. Va na aba "Cloud Messaging"
  3. Role ate "Pares de chaves Web Push"
  4. Clique em "Gerar par de chaves"
  5. Copie a chave publica (comeca com B...)
messaging.usePublicVapidKey("COLE_AQUI_SUA_CHAVE_PUBLICA");
  1. Abra o arquivo firebase-config.js
  2. Substitua "SUA_CHAVE_VAPID_AQUI" pela chave que voce copiou
💡 Dica: As notificacoes so funcionam quando o app esta hospedado online (HTTPS). No computador local nao funcionam.
5
Publicar regras de seguranca
Protege os dados de cada tipo de usuario

🚀 Passo a passo

  1. No Firebase Console, va em Firestore Database
  2. Clique na aba "Regras"
  3. Voce vera um texto padrao. Apague tudo
  4. Abra o arquivo firestore.rules do seu computador
  5. Copie TODO o conteudo e cole no Firebase
  6. Clique em "Publicar"
✅ O que essas regras fazem?
Clientes so veem seus proprios pedidos. Lojistas so gerenciam sua loja. Entregadores so veem entregas disponiveis. Admins veem tudo.
6
Hospedar online e testar
Netlify (gratis) ou Firebase Hosting

🌐 Opcao 1: Netlify (mais facil)

  1. Acesse app.netlify.com/drop
  2. Arraste a pasta delivery-geral inteira para a pagina
  3. Pronto! O app ja esta online com URL propria
  4. Copie o link e teste no celular
Abrir Netlify Drop

🔥 Opcao 2: Firebase Hosting

  1. Instale o Node.js em nodejs.org
  2. Abra o Prompt de Comando e digite:
    npm install -g firebase-tools firebase login firebase init hosting firebase deploy

🧪 Testar o app

  1. Abra o link do app no navegador do celular
  2. Crie uma conta na tela de cadastro
  3. Faca um pedido de teste
  4. Verifique se o pedido aparece no Firebase Console > Firestore Database
💡 Dica: Para criar o primeiro admin, va no Firebase Console > Authentication, copie o UID do usuario, depois va em Firestore > users > [UID] e mude o campo role para "admin".
